📌 Project Overview

The objective of this project was to build a robust, end-to-end data pipeline that simulates real-world sensor data ingestion, performs complex transformations, detects anomalies using Machine Learning, and logs detailed data observability metrics to MLflow.

The pipeline is fully automated and runs within a Dockerized environment using Apache Airflow 3.x, ensuring high availability and professional-grade orchestration.

🏗️ Architecture & Data Flow

The project follows a modern Medallion Architecture pattern to ensure data quality and lineage throughout the lifecycle:

🥉 Bronze Layer Raw Dataset • PostgreSQL Ingestion
▼
🥈 Silver Layer Validated • Enriched • Anomaly Tagged
▼
🥇 Gold Layer Aggregated Metrics • Trend Analysis

1. Bronze (Raw)

Simulated sensor data is appended to raw_sensor_data in PostgreSQL. This layer maintains the immutable source of truth for all readings.

2. Silver (Enriched)

Data is validated with Pydantic. New features like Heat Index and Anomaly Status (via Scikit-Learn) are calculated and merged.

3. Gold (Aggregated)

Final high-level analytics like Trend Slopes, linear regressions, and hourly averages per sensor are computed and stored in processed_sensor_metrics for downstream BI reporting.

🚀 Step-by-Step Implementation

1. Environment Setup & Dockerization ▼

The entire stack is containerized for portability. The docker-compose.yaml initializes five integrated services: Scheduler, Worker, Webserver, Postgres, and MLflow.

💡
Custom Dep Management Utilized the _PIP_ADDITIONAL_REQUIREMENTS env variable to auto-install ML libraries inside the containers without needing a custom Dockerfile.
2. Database Automation & Schema Governance ▼

Implemented a setup_database_schema task. This ensures the environment is self-healing, automatically creating SERIAL, TIMESTAMP, and DOUBLE PRECISION columns on the first run.

CREATE TABLE IF NOT EXISTS raw_sensor_data (
    id SERIAL PRIMARY KEY,
    sensor_id INTEGER,
    timestamp TIMESTAMP,
    temperature DOUBLE PRECISION...
);
3. Incremental Data Simulation ▼

Created a generator representing industrial 24-hour cycles. By using if_exists="append", we simulate an ongoing data stream while preserving historical trends for the AI model.

4. Advanced Transformation & ML Integration ▼
  • Feature Engineering: Calculated the Heat Index using a refined humidity-temperature interaction formula.
  • Linear Regression Slope: Computed the linear slope over the last 24 entries per sensor to detect environmental drift.
  • Isolation Forest: Deployed a 3% contamination model to autonomously flag sensor malfunctions or physical intrusions.
5. Data Observability with MLflow ▼

Every pipeline run is tracked as a unique experiment artifact in MLflow.

📊
Metrics Captured: Detailed statistical distributions (Mean, Std Dev, Min/Max) for all metrics, Dataset Versioning (MD5 hashes), and Audit Artifacts (JSON/CSVs).

📋 How to Run & Monitor

1. Configuration

Ensure your .env file contains the correct Google App Password credentials for the notification system:

.env
AIRFLOW__SMTP__SMTP_PASSWORD=ibfgg
AIRFLOW__SMTP__SMTP_USER=abc@gmail.com

2. Execution & Verification

1. Start the stack: docker-compose up -d.

2. Trigger the etl_mlflow_postgres_pipeline in the Airflow UI (8080).

3. Review the execution trace and health metrics in the MLflow UI (5000).

⚠️
Verification Step Check your Gmail inbox for the ETL Success Report. It should contain a formatted HTML table of the processed sensor metrics.
